# DoD — Надійність комп’ютерних систем (КНУБА)

## Що має бути на здачу (мінімум)
1. **Розрахунки/моделі** (таблиці/формули/пояснення) для кожної задачі.
2. **Звіт PDF**: постановка задачі → вихідні дані → хід рішення → результат → висновки.
3. **Посилання на джерела** (книги/стандарти/статті), якщо використовувались.

## Якість (критерії приймання)
- Формули та позначення визначені (що означає кожна змінна).
- Є перевірка здорового глузду: порядок величин, граничні випадки.
- Результат інтерпретований (що означає для системи/ризику/відмов).

## Типові причини 0 балів
- Лише відповідь без ходу рішення.
- Немає одиниць вимірювання/пояснення параметрів.
- Підміна вихідних даних без обґрунтування.
