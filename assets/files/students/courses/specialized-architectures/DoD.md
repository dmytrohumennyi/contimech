# DoD — Спеціалізовані архітектури комп’ютеризованих систем (КНУБА)

## Що має бути на здачу (мінімум)
1. **Git-репозиторій** з історією комітів по кроках виконання.
2. **README.md**: тема роботи, ПІБ/група, інструкція запуску, версії інструментів.
3. **Звіт PDF** (за ДСТУ 3008-95): хід виконання + ілюстрації + посилання на джерела.
4. **Демонстрація/захист**: коротка доповідь + показ артефактів.

## Якість (критерії приймання)
- Артефакти відповідають методичці (всі обов’язкові пункти виконані).
- Немає «мертвих» файлів: кожен артефакт має пояснення в README/звіті.
- Коміти осмислені (не «update», а що саме змінено).
- Імена файлів та папок відповідають вимогам методички.

## Типові причини 0 балів
- Немає звіту або репозиторію.
- Немає демонстрації.
- Очевидний копіпаст без розуміння (немає пояснень/власних висновків).
